# PowerShell Debugging Approach

## 1. Start the process with monitoring
```powershell
cd C:\Users\cool\Desktop\aiWebSockets
Start-Process -FilePath ".\build-debug\Debug\performance_test.exe"
```

## 2. Find the process ID
```powershell
Get-Process | Where-Object {$_.ProcessName -eq "performance_test"}
```

## 3. Attach debugger or create dump
```powershell
# Get process ID (replace 12345 with actual PID)
$pid = 12345

# Create dump file when hanging
.\build-debug\Debug\performance_test.exe
# When it hangs, in another PowerShell:
C:\Windows\System32\dumpdump.exe -ma $pid C:\temp\performance_test.dmp
```

## 4. Or use Process Explorer
- Download Process Explorer from Microsoft
- Run Process Explorer
- Find performance_test.exe
- Right-click -> "Create Dump" -> "Full Dump"

## 5. Analyze dump in Visual Studio
- File -> Open -> File
- Select the .dmp file
- Debug with Native Only
- Examine call stacks and variables
